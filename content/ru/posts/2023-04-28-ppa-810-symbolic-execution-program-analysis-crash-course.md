## Краткое изложение
В этой лекции Егор Бугаенко разбирает символьное исполнение (symbolic execution) — мощный метод статического анализа, который имитирует выполнение программы, используя символы вместо конкретных входных данных. В отличие от обычного тестирования, где переменной присваивается число (например, 5), здесь она получает символ ($X$). В ходе «выполнения» значения переменных превращаются в математические выражения от этих символов.

Ключевым понятием являются условия пути (Path Conditions, PC). Когда алгоритм встречает ветвление (оператор `if`), он разделяет состояние анализа на две ветви. Для каждой ветви формулируется логическое условие, которое должно быть истинным, чтобы программа пошла именно этим путем. Затем специальный инструмент — SMT-солвер (например, Z3) — проверяет, достижим ли этот путь. Если решение существует, солвер может выдать конкретные входные данные для создания идеального тест-кейса.

Символьное исполнение идеально подходит для поиска трудноуловимых багов: деления на ноль, выхода за пределы массива или утечек памяти. Главная проблема метода — «взрыв путей» (path explosion): в больших программах количество возможных комбинаций ветвлений растет экспоненциально, что делает полный анализ крайне ресурсозатратным.

## Ключевые мысли
* «Символьное исполнение — это не настоящий запуск программы, а симуляция логики через математику».
* «SMT-солвер — это „мозги“ системы, которые превращают программный код в решаемые уравнения».
* «Взрыв путей — наш главный враг; невозможно проверить каждый поворот в сложном лабиринте кода».
* «Автоматическая генерация тестов на основе условий пути — это способ достичь 100% покрытия критических участков».
