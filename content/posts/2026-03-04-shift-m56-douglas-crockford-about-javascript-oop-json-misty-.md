## Summary

In this interview with Yegor Bugayenko, Douglas Crockford delivers a characteristically blunt assessment of the modern programming landscape, focusing on the degradation of JavaScript and the industry's tendency to prioritize familiarity over technical purity. Crockford argues that JavaScript has become a "smelly" and "dirty" language because of a relentless accumulation of "crap" -- features like classes and complex type annotations -- added primarily to satisfy developers who migrated from Java or C and refused to learn the language's original paradigm. He maintains that the "good parts" of JavaScript remain a brilliant functional subset, which he famously describes as "Scheme with dynamic objects," but laments that this core is increasingly buried under redundant, non-orthogonal features.

The conversation shifts to the historical failure of "clean" languages. Crockford cites Smalltalk-80 as perhaps the best-designed language in history, noting it failed not because of its early performance issues, but because it required a fundamental shift in how programmers think. This paved the way for the success of C++, which Crockford describes as an "awful" language that succeeded specifically because it allowed developers to continue writing in old paradigms (C and Fortran) while claiming to use OOP. C++'s survival is a testament to the industry's preference for "transitional" tools that don't demand genuine re-education.

Looking at language evolution, Crockford critiques "Python envy" within TC39. He observes all languages converging into a single, cluttered entity by borrowing features from one another, rather than pursuing distinct design paths. He remains an advocate for minimalism -- developers can still write excellent programs by strictly adhering to the "good parts" and ignoring the industry-mandated "burden."

## Key Insights

- **JavaScript's Core Identity:** "My contention has always been there's a functional subset in JavaScript which is brilliant. It's kind of like Scheme with dynamic objects and that is such a brilliant design."
- **The Burden of New Features:** "The language has gotten a lot dirtier. They just keep adding more and more crap to it... the other stuff is sort of a burden because it gets in the way."
- **The Success of C++:** "It allowed you to be working with an object-oriented programming language without understanding anything about object-oriented programming. You could still write in C, you could still write in Fortran... and people said 'yeah that's the one we want'."
- **Language Convergence:** "There's this tendency we're seeing now -- all of the languages are borrowing features from each other. Eventually they all want to evolve into the same language... that's not really a design path."
- **TC39's Motivation:** "TC39 had a lot of Python envy. They wished that JavaScript were Python... even though at the time JavaScript was much more popular."

## Relevance: 5/5

Crockford provides a masterclass in minimalist philosophy, challenging the industry's obsession with feature accumulation. His historical context regarding Smalltalk vs. C++ offers profound insights into why technically "inferior" tools often win in the marketplace due to social and transitional factors. For any developer questioning the complexity of modern frameworks and TypeScript, this is a foundational critique from one of the industry's most influential figures.
